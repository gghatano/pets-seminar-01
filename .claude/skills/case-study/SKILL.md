# case-study（事例再現の標準手順）

本リポジトリで事例（`case01`, `case02`, …）を再現するときの、実証済みの進め方。
プロジェクトの芯・環境ハマりどころは [`CLAUDE.md`](../../../CLAUDE.md) を前提とする。

## 原則（必ず守る）

- 一次情報（原本 PDF）を確認せずに、事例の事実（事例番号・データ項目・加工方法）を書かない。
- 各記述に出典区分ラベル 〔報告〕〔法令〕〔独自〕〔簡略〕 を付け、混同しない（spec §23.5）。
- `docs/` を単一の正本にする。③の結果は②Notebookの実行出力から作る（二重実装しない）。
- 各ステップの成果物を作るたびに CI ゲート（`CLAUDE.md` 参照）をローカルで通す。

## 手順

### 0. 一次情報の確定
- 対象レポート・事例を `WebSearch` 等で特定（本文取得はプロキシで 403 になりうる。→ ユーザーに PDF 提供を依頼）。
- 原本 PDF を確認（AES 暗号化。`uv run --with pypdf --with cryptography` で `PdfReader(...).decrypt('')` → `extract_text()`）。
- `docs/references/README.md` に書誌情報（資料名・公開主体・時期・URL・参照日・参照箇所）を登録。原本を格納する場合は出典明記。

### 1. 要件整理・事例整理（層①の起点）
- `docs/<case>/00_requirements.md`: 対象事例／レポート要件（元データ項目・加工方法）／制度要件〔法令〕／教材要件／データ要件／トレーサビリティ。
- `docs/<case>/01_case_summary.md`: 事例背景・元データ項目・必要粒度・レポート上の加工方法。すべて出典ラベル付き。

### 2. データモデルと生成器（先に実装して схемаを確定）
- データモデルは図表の構造を尊重して正規化（例: 顧客マスタ／履歴テーブルを結合キーで関連づけ）。
- `src/dummy_data/<case>.py`: seed・基準日固定。加工後分析が成立する人工傾向を作り込む〔独自〕。
- 生成 → `data/<case>/raw/*.csv` にコミット。**再生成がバイト一致**することと、狙った傾向が出ていることを `groupby` で確認。
- `tests/test_<case>_dummy_data.py`: 再現性・スキーマ・PK一意・参照整合性・値域。

> 教材では **読みやすい手書き生成器** を基本とする。vendored の `synthesize` パイプラインは、仕様が複雑・未知でスキーマ推定や評価ループが要る場合に使う（判断は `.claude/skills/README.md` 参照）。

### 3. 設計文書 03〜08（生成スキーマに整合させる）
- 03 加工前テーブル定義 / 04 情報特性・機微度分類 / 05 加工設計（利用目的→必要粒度→方針）/ 06 加工仕様（pandas実装方針）/ 07 加工後テーブル定義＋前後対応表 / 08 Notebook仕様。
- 生成した実スキーマ・実データと必ず一致させる（doc↔data ドリフト防止）。

### 4. Notebook（層②）と結果（層③）
- `notebooks/<case>.ipynb` は `nbformat` で組み立てると壊れにくい。データは GitHub raw URL から取得。
- 段階加工（置換→一般化→削除→維持確認）→前後比較→加工後分析→assert確認テスト、の構成（08準拠）。
- **実行検証**: `DATA_BASE` をローカル `data/<case>/raw` に置換したコピーを `jupyter nbconvert --execute` で回し、0 エラー＋assert セル合格を確認。
- ipynb は ruff format 対象。整形後にもう一度実行検証する。
- `docs/<case>/09_results.md`: 実行済みデータの実数値（前後比較・チェック結果・加工後分析）を反映。

### 5. 仕上げ
- サイト nav（`mkdocs.yml`）に新規ページを追加。README の事例表・Colab バッジを更新。
- 設計文書間・doc↔data↔notebook の整合をレビュー（stale/矛盾なし）。CI ゲートを全て通す。

## 完了の目安

- 別の実装者が設計文書だけで Notebook を実装できる（spec §23.3）。
- Colab で上から実行して完了する。加工前後で何が失われ・維持されたか、利用目的の分析が成立することが分かる。
- レポート記載・法令・独自・簡略が本文で区別されている。
