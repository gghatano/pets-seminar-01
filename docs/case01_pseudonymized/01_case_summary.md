## 1. 事例背景〔報告〕

- **事例名**: 事業者が持つ一つのデータベースに含まれる個人情報を仮名加工情報に加工し利用目的を変更する事例（事例1）
- **事業者**: 食品のオンライン通信販売事業を行う事業者A
- **保有データ**: オンライン通信販売事業に関連して取得した顧客の個人情報（会員属性＋購買・アクセス履歴。§3 参照）
- **従来の利用目的**:
  - お客様がご購入された商品を提供するため
  - お客様が閲覧した当社ウェブページの履歴に基づき、お客様の興味に応じた新商品に関する広告を行うため
- **新たな利用目的（仮名加工情報を作成する背景）**: 新規事業として、特定産地で有機農法により収穫された食品を首都圏の実店舗で販売する事業（実店舗事業）を計画。オンライン通販で取得した個人情報を分析し、ある地域でどのような顧客層（年齢・性別）がどのような商品に関心を有しているかを分析して**出店計画**を検討したい。この目的は従来の利用目的に含まれないため、仮名加工情報に加工して利用目的を変更する。
- **加工後の利用方法**: 顧客の属性類型（年代・性別・居住地域）ごとに購買動向を分析し、出店候補地域の選定に活かす。
  - 加工後の利用目的（公表、図表1-5）: 「お客様の属性（年代・性別・居住地）、商品購入履歴、当社ウェブページの閲覧履歴から、各地域のお客様の興味やニーズの傾向を分析し、実店舗事業の計画を行うため」

## 2. なぜ仮名加工情報か〔報告〕

- 個人単位の特徴を詳細に残したまま比較的簡便に加工でき、探索的な分析（分析手法を事後に柔軟に選択・変更）が可能。
- 統計情報化すると集団傾向しか把握できず分析手法も限定されるため、個人単位分析が必要な本件では仮名加工情報が適する。

## 3. 元データ項目（図表1-1）〔報告〕

| 区分 | 項目 |
|------|------|
| 個人属性情報 | 会員ID、氏名、生年月日、性別、郵便番号、住所、携帯電話番号、電子メールアドレス、クレジットカード番号 |
| 履歴情報（購買に関する情報） | Cookie ID、自社ウェブページのアクセス履歴、購入年月日、購入品目、購入数量、購入金額 |

> 図表1-2 でデータセットを「個人属性情報」と「履歴情報」に分類している。

## 4. 分析上の必要粒度〔報告〕

- 居住地域: 実店舗の商圏内かを判断するため **市区町村単位** での分析を希望。
- 年齢: **10歳区切り程度の年代** が分析できればよい。
- 購入年月日・購入品目: できる限り加工せずに分析したい。

## 5. レポート上の加工方法（図表1-3）〔報告〕

| 項目 | 加工方法 | 加工の型 |
|------|----------|----------|
| 会員ID | 削除し整理番号を付す（分析に不要＋識別禁止義務抵触リスク低減） | 置換 |
| 氏名 | 全部削除 | 削除 |
| 生年月日 | 年齢換算し「10代」〜「80代以上」に置換 | 一般化（丸め） |
| 性別 | 加工しない（生年月日・住所を加工済みのため） | 加工なし |
| 郵便番号 | 削除（加工後の住所を利用するため不要） | 削除 |
| 住所 | 市区町村名までに丸め | 一般化（丸め） |
| 携帯電話番号 | 削除（本人到達性・共用性） | 削除 |
| 電子メールアドレス | 削除（本人到達性・共用性。施行規則31条1号該当時は削除義務〔法令〕） | 削除 |
| クレジットカード番号 | 削除（施行規則31条3号に基づく〔法令〕） | 削除 |
| Cookie ID | 削除（本人到達性、分析に不要） | 削除 |
| アクセス履歴 | 加工しない（キャンペーン反応・閲覧→購入の分析ニーズ） | 加工なし |
| 購入年月日 | 加工しない | 加工なし |
| 購入品目・購入数量・購入金額 | 加工しない | 加工なし |
