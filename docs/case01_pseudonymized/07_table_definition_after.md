> 加工した「後」のスキーマと、加工前後の対応です。顧客の属性は 9 項目 → 4 項目に縮約されます。

| No. | 項目 | データ型 | 形式・値域 | キー | 値の例 |

## 加工前後の対応

| 加工前 | 加工後 | 処理 |
|--------|--------|------|
| 会員ID | 整理番号 | 置換 |
| 氏名 | ― | 削除 |
| 生年月日 | 年代 | 一般化 |
| 性別 | 性別 | 加工なし |
| 郵便番号 | ― | 削除 |
| 住所 | 市区町村 | 一般化 |
| 携帯電話番号 | ― | 削除 |
| 電子メールアドレス | ― | 削除 |
| クレジットカード番号 | ― | 削除 |
| Cookie ID | ― | 削除 |
| 購入年月日/品目/数量/金額 | 同左 | 加工なし |
| アクセス日時/閲覧カテゴリ | 同左 | 加工なし |

直接識別子・本人到達性のある項目は全削除。履歴情報は整理番号で個人単位の結合を維持しつつ、分析に必要な項目をそのまま保持します。
