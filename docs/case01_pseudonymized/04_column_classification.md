# case01（仮名加工情報）: 情報特性の評価

事例 → データを見る → **情報特性** → 加工設計 → 加工仕様 → 実装 → 結果
{ .process-nav }

> 加工方針を決める前の下ごしらえ。各項目を **「単体で個人を特定できるか」「組み合わせで特定につながるか」「結合に必要か」** の観点で見極めます。用語は [用語集](../glossary.md) を参照。

## 見るポイント

- **直接識別子**: 単体で個人を特定できる（例: 氏名）。
- **準識別子**: 単体では特定できないが、組合せで特定につながりうる（例: 生年月日＋性別＋住所）。
- **本人到達性**: その情報で本人に連絡・到達できる（例: メール・電話）。
- **結合キー性**: テーブル・履歴の結合に必要（例: 会員ID）。
- **機微度**: 特定された場合のプライバシー影響の大きさ（加工設計を考えるための補助指標）。

## ざっくりの評価

| グループ | 例 | 特徴 |
|----------|-----|------|
| そのままでは危険 | 氏名・住所・携帯・メール・カード番号 | 直接識別子／本人到達性が高い |
| 組合せで効いてくる | 生年月日・性別・郵便番号 | 準識別子。粗くすれば残せる |
| 結合に必要 | 会員ID・Cookie ID | 識別子。分析には整理番号で代替 |
| 分析に使う | 購入年月日・品目・金額・アクセス履歴 | 識別力は低く、そのまま活かせる |

??? note "customers の項目別評価"
    | 項目 | 分類 | 直接識別性 | 間接識別性 | 機微度 | 結合キー性 | 備考 |
    |------|------|-----------|-----------|--------|-----------|------|
    | 会員ID | 内部管理識別子 | 低 | 低 | 低 | 高 | 作成元との照合で識別禁止義務に抵触しうる |
    | 氏名 | 直接識別子 | 高 | 高 | 中 | ― | 単体で個人識別可 |
    | 生年月日 | 準識別子 | 低 | 高 | 中 | ― | 性別・住所との組合せで識別リスク |
    | 性別 | 準識別子 | 低 | 中 | 低 | ― | 単体の識別力は低い |
    | 郵便番号 | 準識別子 | 低 | 中 | 低 | ― | 住所と重複する地理情報 |
    | 住所 | 準識別子 | 低 | 高 | 中 | ― | 共用性・本人到達性あり |
    | 携帯電話番号 | 直接識別子相当 | 中 | 高 | 中 | ― | 本人到達性・共用性 |
    | 電子メールアドレス | 直接識別子相当 | 中 | 高 | 中 | ― | 本人到達性・共用性 |
    | クレジットカード番号 | 直接識別子相当 | 中 | 高 | 高 | ― | 不正利用で財産的被害のおそれ |
