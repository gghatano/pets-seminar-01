## 4. 教材要件（spec.md 由来）〔独自/簡略〕

- データ加工設計の**思考プロセス**（データ理解→情報特性評価→利用目的→加工設計→実装→確認）を追体験できること（spec.md §24）。
- 3層構成で提示: ① 検討=HTML / ② 実装=Colab / ③ 結果=HTML（spec.md §25）。
- レポート記載・法令・独自設計・簡略化を区別すること（spec.md §23.5）。
- 完結性: 設計成果物だけで別の実装者が Notebook を実装できること（spec.md §23.3）。

## 5. データ要件（ダミーデータが満たすべき条件）〔独自〕

- 元データ構造（R1 の事例）を尊重したデータモデル（顧客マスタ／購買履歴／アクセス履歴等の分割検討）。詳細は [02_dummy_data_spec.md](02_dummy_data_spec.md)。
- 一様乱数にせず、加工後分析が成立する人工的傾向を持たせる（レポート未記載の設定は〔独自〕と明示）。
- 同一人物の複数履歴を生成。乱数 seed 固定で再現可能。
- Colab で短時間に実行できる規模。
