# case01（仮名加工情報）: Notebook仕様書

> デモ Notebook（[② 実装](notebook.md)）の構成。詳細な設計は本サイトの docs を正本とし、Notebook には理解に必要な要点のみ掲載する（spec.md §18）。

## 位置づけ

- 媒体: Google Colaboratory。GitHub 上の生成済みダミーデータ（`data/case01_pseudonymized/raw/`）を raw URL で取得し、ローカル依存なしで上から順に実行できる。
- 役割: 「① 検討」で決めた加工設計を**実際に加工して確認する**（層②）。加工前後比較・確認テスト・加工後分析までを含む。

## セル構成

1. **タイトル・概要**: 対象事例（事例1）、目的、参照レポート、注意事項、docs へのリンク。
2. **事例概要**: 事業者A・保有データ・新たな利用目的・仮名加工の背景（[01](01_case_summary.md) の要約）。
3. **データ取得**: 3 CSV を raw URL から `pd.read_csv`。件数・カラム・サンプル表示。
4. **加工前データの確認**: レコード件数・カラム・型・サンプル。[03](03_table_definition_before.md) へのリンク。
5. **列ごとの情報特性（要約）**: 主要列の識別性・機微度・必要性の要約表（[04](04_column_classification.md) が正本）。
6. **利用目的と必要粒度**: 住所=市区町村、年齢=年代（[05](05_processing_design.md) の要約）。
7. **加工方針（要約）**: 図表1-3 相当の方針表。
8. **加工処理**（段階実行、[06](06_processing_spec.md) 準拠）:
   1. 整理番号の付与と会員IDの置換（3テーブル）
   2. 生年月日→年代 の一般化
   3. 住所→市区町村 の一般化
   4. 不要・直接識別列の削除（氏名・郵便番号・携帯・メール・カード番号・Cookie ID）
   5. 履歴情報の維持確認
   各ステップで「何を・なぜ・コード・結果」を提示。
9. **加工前後比較**: カラム構成・サンプル・削除/一般化/維持の対比。
10. **加工後データによる分析**: pandas `groupby` 中心。市区町村×年代×性別 の購買傾向、地域×平均購入金額 など。
11. **確認テスト（assert）**: 削除列の不在／整理番号の存在・一意対応／レコード数不変／同一人物の履歴関係維持。
12. **まとめ**: データ理解→情報特性→利用目的→加工→結果→利用 の振り返り。

## 実装原則

- 加工の中核処理は Notebook 上で読めるようにする（過度に関数へ隠蔽しない）。住所一般化など一定のロジックは関数化してよいが内容は Notebook 上で確認可能に。
- pandas 2 系互換で記述（Colab 実行環境）。
