# case01（仮名加工情報）: ダミーデータ仕様書

> 対象: 事例1（食品オンライン通販の事業者A）。元データ構造は [事例整理](01_case_summary.md) の図表1-1 に基づく。
> 本仕様のデータモデル・件数・分布はすべて教材用の**独自設定〔独自〕**（レポート未記載）。
> 実装: [`src/dummy_data/case01_pseudonymized.py`](https://github.com/gghatano/pets-seminar-01/blob/main/src/dummy_data/case01_pseudonymized.py)

## 1. データモデル〔独自〕

図表1-2 の「個人属性情報／履歴情報」の分類を尊重し、3テーブルに正規化する。会員ID で関連づける。

| テーブル | 論理名 | 役割 | 主キー | 外部キー | 1レコードの粒度 |
|----------|--------|------|--------|----------|-----------------|
| `customers` | 顧客マスタ | 会員の属性情報 | 会員ID | — | 顧客1人 |
| `purchases` | 購買履歴 | 購入トランザクション | 購買ID | 会員ID → customers | 購入1件 |
| `web_access` | Webアクセス履歴 | 自社サイト閲覧ログ | アクセスID | 会員ID → customers | アクセス1件 |

図表1-1 の全項目を割り当てる（属性→customers、購買/アクセス→purchases・web_access）。Cookie ID は端末に紐づく想定で web_access に置く。

## 2. データ件数〔独自〕

| 項目 | 値 |
|------|----|
| 顧客数 | 800 |
| 1顧客あたり購買件数 | 平均 約6件（Poisson(5)+1） |
| 1顧客あたりアクセス件数 | 平均 約10件（Poisson(9)+1） |
| purchases 総件数 | 約4,800 |
| web_access 総件数 | 約8,000 |

Colab で短時間に実行できる規模。

## 3. データ分布（人工的傾向）〔独自〕

加工後に利用目的（地域×顧客層×商品関心）の分析が成立するよう、以下の傾向を持たせる。

- **地域**: 首都圏（東京・神奈川・埼玉・千葉）の15市区町村に重み付きで分布。実店舗の商圏フラグを内部的に保持。
- **年代 × 商品カテゴリ**: 若年層は惣菜・菓子・飲料寄り、高年層は野菜・鮮魚・米・有機寄り。
- **地域 × 購入金額**: 東京はやや高単価（係数1.15）。
- **年齢**: 平均45・標準偏差15（18〜84歳）。性別は男性48% / 女性52%。
- 商品カテゴリ（10種、食品通販想定）: 野菜・果物・精肉・鮮魚・米・穀物・惣菜・飲料・菓子・調味料・有機食品セット。

これらは教材用の人工設定であり、実際の統計を表すものではない。

## 4. 同一人物の履歴〔独自〕

各顧客に複数の購買・アクセス履歴を生成し、会員ID で結合可能。加工後も整理番号で個人単位の履歴分析ができる構造とする。

## 5. 再現性〔独自〕

- 乱数 seed 固定（`SEED = 20260717`）、年齢・履歴期間の基準日固定（`REFERENCE_DATE = 2026-07-01`）。
- 使用ライブラリは標準・numpy・pandas のみ。
- `uv run python src/dummy_data/case01_pseudonymized.py` で再生成（バイト一致）。生成結果は `data/case01_pseudonymized/raw/` にコミットし、Notebook は生成済みデータを取得する。

- `data/case01_pseudonymized/raw/purchases.csv`
- `data/case01_pseudonymized/raw/web_access.csv`
