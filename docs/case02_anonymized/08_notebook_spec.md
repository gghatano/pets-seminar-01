# case02（匿名加工情報）: Notebook 仕様

> 開発者向けメモ。Colab Notebook（`notebooks/case02_anonymized.ipynb`）の構成。学習の本筋は [① 事例概要](01_case_summary.md) から。

## 構成（12ブロック）

1. タイトル・所要時間・ゴール・匿名加工の流れ（5ステップ）
2. データ取得（GitHub raw URL、顧客属性・取引情報・購買履歴）
3. 加工前の確認（件数・スキーマ・特異値のプレビュー）
4. 仮IDへの置換（34条3号。対応表は保存しない）
5. 直接識別子の削除（氏名・電話番号）
6. 準識別子の丸め（生年月日→年代7区分、住所→市区郡、利用日時→分単位）
7. 特異な記述の処理（商品名→カテゴリ、数量の上限丸め、金額→区分。34条4号）
8. 不要IDの削除（店舗ID・取引ID・担当者ID・商品ID・明細ID）
9. 加工前後の比較（項目数の縮約、特異値が消えたこと）
10. 匿名性の確認（元IDに戻せない／(年代×性別×市区郡)群の大きさ＝k-匿名性の目安）
11. 加工後分析（属性 × 購買傾向：年代別カテゴリ、エリア別金額区分）
12. まとめ＋③ 結果ページへのリンク

## 方針

- docs を正本とし、Notebook には理解に必要な要点のみ掲載する。
- データは生成済み CSV を raw URL から取得（ローカル依存なし）。pandas 2 系互換で記述。
- 確認テスト（assert）: 削除列の不在・仮ID化・秒情報の除去・特異値の除去・件数維持。
- 実行検証は `DATA_BASE` をローカル `data/case02_anonymized/raw` に置換して `nbconvert --execute`。
