## 匿名加工情報とは（先取り）

特定の個人を識別できず、かつ**元に戻せない**ように加工した情報です。case01 の**仮名加工情報**と目的が対照的です。

| | 仮名加工情報（case01） | 匿名加工情報（case02） |
|---|---|---|
| ねらい | 個人単位の分析は**残す**（社内利用が主） | 再識別・連結を**断つ**（第三者提供・公開も想定） |
| 加工の強さ | 比較的ゆるい | **厳しい**（一律の作成基準あり） |
| 「同じ人の履歴」 | 整理番号で**たどれる** | **たどれないようにする** |

## case02 で扱う予定の考え方

case01 の型（① 検討 → ② 実装 → ③ 結果）を踏襲しつつ、匿名加工情報ならではの論点を加えます。

- **作成基準**（識別できる記述の削除・個人識別符号の削除・**連結符号の削除**・**特異な記述の削除**・データベースの性質に応じた措置）。
- **再識別リスクを下げる技術**（準識別子の組合せ対策・**k-匿名性**の考え方・外れ値／希少値の処理・より強い一般化）。
- 匿名加工情報に関する**公表**や**識別行為の禁止**などの取り扱い。

制度の詳細は、公開時に一次情報（個人情報保護委員会の事例レポート・ガイドライン）に沿って整理します。
